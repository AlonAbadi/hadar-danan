## 6. מטריצת S6 (הכלל + 4 תאים)

- **יש לקוחות×המסר:** *"יש כבר אנשים שמשלמים לכם — הוכחה שהערך אמיתי. אבל סימנתם שהמסר עוד לא ברור: כשהערך קיים והמסר מטושטש, אתם נמכרים למרות המסר, לא בזכותו. מה שחסר זה לא עוד שכנוע — משפט אחד שיגיד את מה שכבר עובד בשקט."*
- **תחילת-דרך×הזמן:** *"אתם בתחילת הדרך וסימנתם שהזמן עוצר — ושתי אלה יחד מספרות משהו: זה לא שאין לכם מה להגיד, זה שכל דקה הולכת להרצה במקום למסר. בשלב הזה משפט חד שווה יותר מעשר שעות עבודה."*
- **מוכרים-תקוע×המחיר:** *"אתם כבר מוכרים אבל תקועים, וסימנתם את המחיר. הפער עדין: כשמדברים על מחיר, בדרך כלל המסר עוד לא עשה את העבודה של להסביר למה דווקא אתם. מחיר מרגיש גבוה רק כשהייחוד לא נשמע."*
- **זרם-קבוע×התוכן:** *"יש לכם זרם לקוחות קבוע — הבסיס איתן. אבל סימנתם תוכן, ודווקא בשלב הזה זה הגיוני: כשהעסק עובד מפזרים תוכן לכל כיוון ומאבדים את החוט. יותר תוכן לא יחזק — משפט אחד שכל סרטון חוזר אליו כן."*

פרומפט: *"חלץ ראיות מסחריות מכל התשובות שניתנו (5-6). `has_paying_clients` דורש ציטוט מילולי חיובי; כסף במסגרת-חוסר ('אין כסף','בקושי סוגרת') → distress_money=true, לא ראיה. שאיפה עתידית אינה ראיה. החזר JSON בלבד."*
**ציון (דטרמיניסטי):**
```
raw = 0.45*has_paying_clients + 0.20*price_points + 0.20*min(years/3,1) + 0.15*min(team/2,1)
depth_conf = (chars<450 ? 0.85 : 1.0)          // עומק = ביטחון בלבד, לא אות-כסף
evidence = raw * depth_conf * (distress_money ? 0.5 : 1.0)
```
**regex בוסטר/סינון:** STAFF (`יש לי עסק|צוות|\d+ עובד(ים|ות)` **וגם לא** `לא עובד`) = +0.15 (המפריד החזק 3.1x). MONEY (`₪|אלף|מחזור|הכנסה` **וגם לא** ליד `אין|בלי|חסר`) = +0.08. "לקוחות" לבד = רעש.
**טבלת-אמת:** A) HIGH+evidence≥0.72+טלפון → **קונסיירז'** (המסלול הפרמיום היחיד). B) LOW+חלש → **כוורת ₪590**. C) HIGH+חלש → strategy-warm (תור-סקירה, לא פגישה-אוטו). D) LOW+חזק → strategy-warm+דגל (re-scan שבועי).
**רף 0.72 → ~4-5/שבוע צפוי** (7%×60; הטענה "9" הייתה טעות חשבון). תקרה קשה 10/שבוע. **שבוע-1 מפושט (הכרעה 6):** בלי מונה-קיבולת חי ובלי רף 0.80 — קופי-נדירות סטטי, ספירה ב-/admin/today, בוסטרי-regex כעמודות-צל בלבד. הנוסחה הקנונית המחייבת (בוסטרים-בתוך-raw, תקרה 1.0, hpc בדליים, עומק פר-תשובה) — בסעיף ג' של ה-AUDIT.

## 9. תמחור + מנגנון הקיזוז בקארדקום (האתגר)
**האבחון:** ה-SumToBill ושורת-החשבונית קורסים לאותו נטו → החשבונית מציגה ₪490 כמחיר הסדנה במקום ₪1,080+הנחה → **דיווח מע"מ שגוי**; אין `credited_toward` → double-spend; ושתי מערכות-קרדיט חיות.
**הפתרון:**
1. **סולם סטטי** ב-`lib/products.ts` (רק מוצרים חד-פעמיים; מנוי-חוזר לא מקזז):
```ts
CREDIT_LADDER = {
 workshop_1080:[{from:"signal_hive_590",amount:590},{from:"challenge_197",amount:197}],
 course_1800:[{from:"workshop_1080",amount:1080},{from:"signal_hive_590",amount:590}],
 strategy_4000:[{from:"course_1800",amount:1800}],
}  // קיזוז אחד (הגבוה), לא נערם
```
2. `lib/credit-ladder.ts` → `computeCredit(userId,target)`: שולף purchases completed שהם `from` **ו-`credited_toward IS NULL`**, max, מוגבל למחירון. **הלקוח לא שולח סכום** (POST `{product,user_id}` בלבד). `net=list−credit`.
3. **net≥1:** LowProfile net + **שתי InvoiceLines:** שורה `סדנה יום אחד ₪1,080` + שורה `זיכוי בגין רכישת כוורת האות — חשבונית #<src> −₪590`. **לא "תשלום חלקי".** מע"מ על ה-₪490 (הנחה אמיתית מקטינה בסיס).
4. **net≤0:** לא לפתוח סשן קארדקום — להעניק ישירות (`grantProduct`, purchase completed, net_paid=0, credit_applied=list) + קבלת ₪0 עם שורת-הנחה. שלילי → cap ל-0, בלי החזר. **מינימום קארדקום:** net<~₪5 → כמו net≤0.
5. **אטומי:** `UPDATE src SET credited_toward=$upsell WHERE id=$src AND credited_toward IS NULL` (rowCount===1) — **בשלב שמסמן completed** (webhook/grant), לא ב-checkout-start.
6. **ניקוי:** למחוק `getUserCredit`/`CreditBanner` מ-workshop/challenge. הסולם המבוקר יורש (מיישב את "הקרדיט הוסר").
7. **אינווריאנט:** `gross = credit_applied + net_paid`; שמור `amount=gross, amount_paid=net`.

## 11-ב · תפרי מנוע-הפער (לבנות את המשפך מוכן-לקבל)
מנוע-הפער (`lib/signal/gap-detect.ts`, מיגרציה 059, `GAP_ENGINE_ENABLED=false`) הוא **מסלול נפרד** — שכבת-הנשמה, מאומתת פנימית (AC1=0.73, רצפת-מצוקה 100%) אך **חסומה לחשיפה למשתמש עד שער-אימות אנושי** (רב/משפיע + מורה של ימימה, אפס "מזיק"). **לא לצמד את לו"ז שני המסלולים.** בונים את המשפך *מוכן-לקבל* דרך שני תפרים מוגדרים:
