## התובנה שהוכחה אמפירית (על 45 אבחונים אמיתיים)

בניתי גלאי דטרמיניסטי (חוקים בלבד) והרצתי על 45 האבחונים מול סט-הזהב (R2):

1. **רצפת המצוקה בחוקים בלבד: recall 3/3, אפס false-positives.** וטו-מילון זול תופס את כל הכאב הטרי. **זה נשלח כשכבה 0.**
2. **הכרעת הפער בחוקים בלבד: טועה-בביטחון על ~1/3.** על מקרי "הכתרה-כאילו-הגיע" (L18: Q3 עם "למדתי" אבל Q5 "היום אני חייבת ואהיה מספר 1... אתחיל ללמד" = overshoot) — הכללים רואים "למדתי" ואומרים metabolized. **זה מוכיח שההכרעה חייבת להיות ה-LLM ensemble.**

לכן הארכיטקטורה: **חוקים לרצפת-המצוקה ולהימנעות-דקיקה בלבד; LLM ל-seam; הימנעות על כל אמצע.**

## איך הצינור עובד (הימנעות-כברירת-מחדל)

```
5 תשובות
 Stage 0  crisis floor (חוקים)      → כאב טרי? כן → safety=do_not_name, abstain. עצור.
 Stage 0.5 thin?                     → פחות מ-60 תווים ב-Q3+Q5? → abstain. עצור.
 Stage B  seam ensemble (Sonnet)     → 3 דגימות × 3 מסגורים (ישיר/יריב/NLI) = 9 קריאות
 Stage C  fusion (ערבוב)             → במקביל
 Stage E  gate מכויל                 → הסכמה חזקה metabolized → "אין פער" (אַשֵּׁר מתנה)
                                       → הסכמה חזקה overshoot (≥70%) + עוגן → מועמד
                                       → כל אמצע → abstain
 Stage F  generation מוגבל           → סף+מעבר, בנוי אך ורק מציטוטים מאומתים
 Stage G  quote-or-discard           → כל טענה חייבת ציטוט מילולי (בדיקת שחזור); אין → abstain
```

עקרונות-הברזל אכופים בקוד: המתנה כבר קיימת (לעולם לא "חסר"); מסגור יריב שנלחם ב-prior של "תמיד למצוא פער"; ציטוט-או-זריקה כאינvariant; רצפת-מצוקה כווטו קשיח; **בספק — abstain**.

## איך מפעילים (הסדר הבטוח, מ-SOLUTION)

1. **אימות אנושי (חובה לפני חשיפה):** רב/משפיע חב"די + מורה מוסמכת של שיטת ימימה + הדר מדרגים במסך `/admin/signal/gap` (או בערכה שבנינו). תנאי מעבר: ≥85% מדויק-או-קרוב, **אפס "מזיק"**, recall רצפת-מצוקה כמעט-מושלם.
2. **Phase 0 — פנימי בלבד:** `GAP_ENGINE_ENABLED=true` ב-preview → מריצים `node scripts/gap-eval.mjs` → הקריאות נשמרות לעמודות הפנימיות → סוקרים במסך. **המשתמש לא רואה כלום.**
3. **canary:** רק אחרי שהשער נעבר — חשיפה זעירה, confirm-first ("האם זה מדייק?"), kill-switch חי (הדגל).
4. **מהדקים-בלבד:** הסף זז רק לכיוון "מציג פחות".

השער הזה הוא בקוד: מסך הסקירה מציג דשבורד עם ספירת "מזיק" (שער קשיח) ו"מדויק+קרוב".

**מתוקף (על דאטה אמיתי):**
