# משימה ל-Claude Code: הטמעת עמוד כוורת האות מהמוקאפ המאושר

## שלושת הקבצים שקיבלת

1. beegood-kaveret-haot-master.html - מקור האמת. לא רפרנס, מקור. שים אותו ב-design/kaveret/ בריפו.
2. kaveret-design-spec.md - תמצית הערכים והחוקים. בכל סתירה - קובץ ה-HTML קובע.
3. visual-check.mjs - שער האיכות. שים ב-scripts/.

## חוקי עבודה מחייבים

1. אתה מעביר, לא מעצב. אפס החלטות עיצוב עצמאיות. כל ערך CSS מועתק מילולית מהמוקאפ: hex, פיקסלים, גרדיאנטים, צללים, עקומות אנימציה, delays. אם ערך לא ניתן לביטוי מדויק ב-Tailwind - כתוב CSS רגיל ב-CSS Module. אסור להחליף ערך בקירוב של Tailwind.

2. מבנה: ראוט app/kaveret (או הראוט הקיים של עמוד המוצר), קומפוננטה ראשית + CSS Module שמכיל את ה-CSS מהמוקאפ כמעט אחד-לאחד. ה-JS מהמוקאפ עובר ל-hooks עם אותה לוגיקה בדיוק: מתמטיקת הנקודות מ-scrollLeft (לא IntersectionObserver בקרוסלה), הזזת הגלולה ב-translateX שלילי, navigator.share עם נסיגה, clipboard עם נסיגה.

3. דאטה: התוכן נטען בשאילתת Supabase אחת ב-server component לפי userData.id (public.users.id - לעולם לא authUser.id). הטקסטים, האות, הכרטיסים והתסריטים מגיעים מהרשומה; המבנה הוויזואלי לא משתנה לפי תוכן.

4. אל תיגע בשום דבר אחר בריפו: לא ב-DNS, לא ברשומת ה-TXT של Google Search Console, לא במיגרציות קיימות.

5. סגנון קוד: בלי אימוג'י, בלי מקפים ארוכים, הערות בעברית או אנגלית פשוטה.

## לולאת האימות - חובה, לא המלצה

התקנה חד פעמית:
npm i -D playwright pixelmatch pngjs && npx playwright install chromium

אחרי כל סבב שינויים משמעותי, וכתנאי לסיום:
node scripts/visual-check.mjs design/kaveret/beegood-kaveret-haot-master.html http://localhost:3000/kaveret

המשימה לא גמורה עד שכל חמש נקודות הבדיקה מחזירות PASS (הפרש קטן מ-1.5%). אם נקודה נכשלת: פתח את visual-diffs/diff-N.png, זהה את האזור השונה, תקן לפי הערך המדויק במוקאפ, הרץ שוב. אל תרחיב את הסף ואל תדלג על נקודה.

## הבדלים מותרים בלבד

- שם הלקוחה, האות והטקסטים מגיעים מדאטה במקום להיות קשיחים.
- חשיפת המילים בלוח רצה רק בביקור ראשון (sessionStorage key: kaveret_reveal_seen).
- קישור הוואטסאפ מקבל את המספר האמיתי ממשתנה סביבה.
כל הבדל אחר מהמוקאפ = באג.
