## ב. גל 4 (תשלומים) — נבנה מחדש

1. **שתי שורות-חשבונית = יכולת לא-מוכחת.** הקוד היום שולח שורה אחת בלבד (`InvoiceLines.*`, checkout:266-279, lowprofile:77-88), ותחביר `InvoiceLines2.*` לא מופיע בשום מקום. **ואם יצירת החשבונית נכשלת — החיוב עובר בלי חשבונית** (webhook רק מלוגג, :823-829). לכן:
   - **שלב א' (בטוח, ללא יכולת חדשה):** שורה אחת שהתיאור נושא את שובל-הביקורת: `"סדנה יום אחד ₪1,080 בקיזוז ₪590 (חשבונית #<src>), לחיוב ₪490"`, Price=net. מע"מ זהה ונכון.

2. **טענת המע"מ מתוקנת:** המע"מ היום **נכון חוקית** (מע"מ על מחיר אחרי הנחה אמיתית; "תשלום חלקי" דווקא היה מכפיל מע"מ על ה-590). הבעיה היא תיאור מטעה ושובל-ביקורת, לא דיווח שגוי.
3. **למחוק מגל 4:** ענף net≤0, grantProduct, קבלת ₪0, רצפת ₪5 — לפי הסולם עצמו הנטו המינימלי הוא **₪490** (קוד מת), ומוצר ה-test ₪1 מוכיח שאין רצפת-Cardcom.

(הערה: CLAUDE.md "credit system removed" — לא נכון; getUserCredit חי ב-workshop+challenge. המפרט צדק.)

## ג. הניתוב — נוסחה קנונית אחת (מחליף את חלק 7)

```
llm = sonnet_extract(answers)   # temp 0, JSON; הפרומפט: "מכל התשובות שניתנו (5-6)" — לא "5"

hpc   = quote-חיובי-קיים ? 1.0 : (מרומז ? 0.5 : 0.0)     # דלי דטרמיניסטי, לא float של LLM
years = clamp(llm.years ?? 0, 0, 40)                      # חשבון-שנים בשרת, לא LLM
team  = llm.team_size ?? 0

# regex = fallback בלבד (נורה רק אם ה-LLM פספס) — מונע ספירה-כפולה של הסיגנל החזק
staff_boost = STAFF_RE && team==0 ? 0.15 : 0
money_boost = MONEY_RE && !llm.price_points ? 0.08 : 0

raw = 0.45*hpc + 0.20*price_points + 0.20*min(years/3,1) + 0.15*min(team/2,1)
raw = min(raw + staff_boost + money_boost, 1.0)           # בוסטרים בתוך raw + תקרה קשה

answered   = מספר תשובות שנענו                            # דילוג-Q4 לא נענש
depth_conf = total_chars < 90*answered ? 0.85 : 1.0
evidence   = raw * depth_conf * (distress_money ? 0.5 : 1.0)
```

**סדר-הכרעה מחייב:** crisisFloor → בדיקת קיום מפתח-1 → טבלת-אמת → בקשת-טלפון-פעם-אחת → תפיסת-slot אטומית.

**תיקונים מחייבים:**
- **"~9/שבוע" = טעות חשבון פי 2.** הנגזר מהדאטה: 7%×60 ≈ **4-5/שבוע** (בערך שווה לקיבולת). לתקן ולמדוד בשבועיים הראשונים.
- **תקרת המומחה-סולו:** בלי צוות ובלי שפת-כסף המקסימום המבני הוא 0.65 < 0.72 — מטפלת/עו"ד ותיקים לעולם לא יגיעו אוטומטית לקונסיירז'. **לא לתקן במשקלים** (המפרידים המאומתים הם אלה); במקום: תיוג `solo_premium_candidate` (HIGH + years≥5 + hpc=1.0 + עמוק + evidence∈[0.55,0.72)) בראש תור strategy-warm, **וכלל years≥7 גובר על מפתח-1 LOW** (ותיק שממעיט בעצמו לא ייפול לעוגן ₪590).
- **פסיקות-קצה:** אין מפתח-1 → לעולם לא לנתב (partial_flow + מייל-השלמה). retakes: המסלול לפי האחרון-השלם, קונסיירז' מונוטוני 30 יום, slot אחד/משתמש/שבוע. פרמיום-בלי-טלפון: בקשה פעם אחת במסך-הסוף; סירוב → strategy-warm; **slot לא נתפס עד שיש ערוץ**. distress+עסק-חזק → C עם דגל `distress_strong` לחילוץ ידני.
- **פישוט שבוע-1:** בלי מונה-קיבולת חי ובלי החלפת-קופי-במלואו (ב-4-5/שבוע מול תקרה 10 זה לא יורה) — קופי-נדירות סטטי + שאילתת-ספירה ב-/admin/today. רף 0.80 באמצע-שבוע = קוד מת — למחוק או להמיר ל-carryover. בוסטרי-regex כעמודות-צל בלבד ל-100 האבחונים הראשונים. **כוונון-רף:** חלון 30 תוצאות אחרונות, הידוק רק ב-≥5/30 couldn't-pay, הרפיה רק ב-≤1/30, מהלך אחד מקסימום ל-6 שבועות — אחרת מכווננים על רעש.
